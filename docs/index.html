Problem / Data

Heavy-tailed target, missingness, and market heterogeneity

Why the problem is hard

Used-car prices are highly right-skewed and segment-dependent. A single RMSE number can hide meaningful error differences across price ranges and subgroups. This motivates both nonlinear modeling and richer diagnostics.

The data also mixes numeric signals such as year and odometer with noisy categorical fields such as condition, fuel, title status, state, and manufacturer. That combination makes both preprocessing and post-hoc interpretation part of the modeling problem instead of an afterthought.

426,880 raw rows Craigslist dataset Leakage-aware split

What the team had to build

The project required cleaning large raw listings, engineering mixed-type features, training multiple baselines, comparing nonlinear alternatives, exporting visual diagnostics, and packaging a runnable demo.

Why split discipline matters

A careless random split can let duplicated market patterns leak across train and test. The repository keeps the evaluation story anchored in a leakage-aware workflow so metric comparisons stay meaningful.

What the upgrade adds

The extension does not replace the original project. It adds a second interpretability layer showing where the broad-domain model is reliable and where subgroup or price-band caution is needed.

Modeling

Comparative modeling and a strong nonlinear baseline

The repository keeps linear baselines (OLS/Ridge/Lasso), text-augmented Ridge, and boosted trees in one consistent pipeline. The nonlinear model leads on broad-domain fit while preserving reproducibility and exportability.

This is useful pedagogically because the project does not jump straight to one high-scoring model. It shows the full modeling ladder: interpretable linear baselines, a text-aware regression baseline, and then a nonlinear learner that better captures mixed-feature interactions.

Why the upgrade matters

The original repository already identified a strong nonlinear regressor. This upgraded site adds the missing second layer: where the model is reliable, where error scales up, and how the same predictor should be read differently across price ranges and subgroups.

That makes the project look more like a deployable analytics system and less like a single leaderboard snapshot.

Baseline stage

OLS, Ridge, and Lasso establish the minimum viable regression story and expose how much linear structure the engineered table can explain on its own.

Feature-extension stage

Ridge + TF-IDF tests whether free-text information from listings adds predictive value beyond the structured fields.

Nonlinear stage

HistGradientBoosting becomes the best broad-domain model because it handles heterogeneous interactions more naturally than the linear baselines.

    What this static demo can prove

    It proves the repository already has a coherent end-to-end path from cleaned inputs to a prediction-facing interface. It also demonstrates how the GitHub Pages version can preserve the same narrative without server-side inference.

    What it cannot prove

    Because outputs are precomputed, the GitHub Pages site is a communication artifact, not a live benchmark. Real runtime behavior still belongs to the local Streamlit app and trained model export.

    Why keep both versions

    The static site is ideal for sharing and grading, while the local app remains the faithful runnable demonstration of the original project deliverable.

    Advanced Extension

    Subgroup and decile diagnostics added in this upgrade

    The extension adds diagnostics beyond aggregate metrics: subgroup MAE, decile error profiles, calibration-style views, and age-bucket behavior. These plots are available in detail on the Results page.

    Decile drift is explicit

    Error is not uniform across the market. High-price listings show much larger absolute-dollar miss even when the overall fit remains strong.

    Subgroups are uneven

    Condition, fuel type, and vehicle age each shift error behavior. The upgrade makes those segment effects visible instead of hiding them inside one MAE.

    Deployment reading changes

    The practical recommendation is to show prediction + segment context together. That is the main value added by this upgraded companion site.

    Limits and Use

    What this project should and should not be used for

    What it does well

    It offers a strong course-project example of data cleaning, tabular regression, baseline comparison, nonlinear modeling, error analysis, and deployment-minded packaging around a realistic marketplace dataset.

    The upgraded diagnostics also make the model's failure profile visible enough to support more careful interpretation.

    What remains out of scope

    This is not a production-grade valuation engine. It does not model macroeconomic shifts, dealer-specific strategy, geographic liquidity, or confidence intervals calibrated for commercial risk decisions.

    The correct reading is: strong educational system, useful analytical prototype, not a finalized market pricing product.
