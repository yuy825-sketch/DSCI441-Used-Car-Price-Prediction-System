Problem Description / Motivation

Why used-car price regression is not a single-metric task

Price prediction must handle heavy tails, missingness, and nonlinear segment interactions. A single global RMSE can hide large differences across price ranges and vehicle segments.

  • Target is strongly right-skewed.
  • Many categorical fields are sparse/missing.
  • Error scale increases with true price (heteroskedasticity).

Dataset Description

Craigslist listings with mixed tabular structure

The dataset contains 426,880 raw listings with numeric, categorical, and optional text fields. Cleaning and feature derivation produce a stable training table for fair model comparison.

Experiments / Model Selection

Linear baselines, text-augmented ridge, and boosted trees

Experiments include OLS, Ridge, Lasso, Ridge+TF-IDF, and HistGradientBoosting. The nonlinear model gives the strongest broad-domain fit while preserving a reproducible pipeline.

Why nonlinear wins: it captures segment interactions that linear baselines only approximate.

Why baselines stay: OLS/Ridge/Lasso still anchor interpretation and set a reproducible floor.

Bottom Note

What this poster prioritizes

The focus is the full project arc: realistic data cleaning, model comparison, nonlinear gains, and the extra diagnostic layer needed to interpret pricing error instead of only reporting one leaderboard score.

Evaluation

Global fit is strong, but error behavior is segment-dependent

The best model achieves high broad-domain fit, yet error distribution differs by price decile and subgroup. Extension diagnostics expose these deployment-relevant patterns.

  • The headline model is strong enough to summarize the broad market, but not uniform enough to justify one confidence message.
  • High-price listings remain the most visible source of larger absolute-dollar miss.
  • The extension therefore shifts the project from score reporting to decision-aware evaluation.

Broad-domain best: R² 0.8061, RMSE 6649, MAE 3384.

Decile drift: MAE rises from about $1.8k to about $10.1k (D1 -> D10).

What the figures show

The model is useful overall, but it compresses some market regions and loses precision in the expensive tail.

Why that matters

A user-facing system should report more than one scalar score when uncertainty is segment-dependent.

Deployment and Interpretability

From offline metrics to user-facing diagnostics

The Streamlit app provides an input -> predict interface. The extension recommends pairing predictions with subgroup/decile diagnostics so users can interpret uncertainty by market segment.

In other words, the poster is not claiming a fully production-ready valuation engine. It shows that the course project now includes the right packaging layers: reproducible training, comparative evaluation, demo exposure, and a clearer reading of failure modes.

Recommended production view: show global metrics and segment-aware diagnostics together.

Future direction: segment-specific models and interval predictions for risk-aware pricing.

  • The GitHub Pages companion turns the same message into a static, deployable walkthrough.
  • The poster, report, and website now tell the same core story across formats.

How the upgraded diagnostics should be used

The main contribution of the extension is not a new winning model; it is a better interpretation protocol for using the same predictor responsibly in a user-facing setting.

  • Show the point estimate together with a segment-aware uncertainty note.
  • Treat premium listings as higher absolute-dollar risk cases even with good global R².
  • Use subgroup plots to flag where future segment-specific models may be worth the cost.
